# HFCC Recruiter / Hiring Manager Brief

## 30-Second Summary

HFCC is a public PostgreSQL/Supabase architecture proof project showing
database-first backend design for complex product systems, including
RLS, event-driven workflows, jobs, ledger modeling,
commerce/subscription structures, messaging, and audit logs.

## Why This Matters

HFCC shows senior backend thinking beyond CRUD. The repository focuses
on correctness, auditability, workflow modeling, data ownership, domain
complexity, and operational systems.

The project demonstrates how a backend engineer thinks about
product-system foundations: durable state, permission-aware data access,
async processing, financial movement, and traceable operations.

## Relevant Roles

- Senior Backend Engineer
- Backend Systems Engineer
- Platform Engineer
- Product Systems Engineer
- Technical Lead
- Staff-leaning Backend Engineer
- ERP / Marketplace / Fintech-adjacent Backend Engineer

## Signals to Notice

- PostgreSQL-first design
- RLS-aware modeling
- Event-driven architecture
- Ledger principles
- Domain type registry
- Workflow/state modeling
- Auditability

## What This Does Not Claim

- Not proof of massive scale
- Not a production SaaS by itself
- Not a replacement for private commercial work
- Not a compliance-certified financial system
